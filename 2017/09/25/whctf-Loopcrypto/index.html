whctf Loopcrypto

发表于 2017-09-25 | 分类于 Android

在霸哥的带领下，做了whctf的Loopcrypto这道题，学习了下apk的动态调试，收获颇多。

0x01 程序分析

反调试

用jeb反编译apk可以看到程序加载了libcheck.so库，并调用了里面的check函数，参数为该apk的签名以及我们输入的字符串。刚开始调试so库的时候（没用debug模式启动应用），IDA attach上去运行，程序就退出，原因出在libcheck.so库的init_array里定义了一个函数sub_83DC：

遇到fork和ptrace都会断开调试，想过用patch so库重打包的方式绕过反调，但check函数里有对apk签名作校验，所以还得patch java层代码，并不会smali……所以这里通过动态patch的方式，直接修改PC寄存器绕过反调代码。

函数注册

用IDA查看libcheck.so库的导出函数，看到了JNI_Load函数，说明check函数是通过动态注册的。

通过分析，可以发现通过JNI_Onload注册的check函数为sub_87FC：

对sub_87FC函数分析，发现其调用的sub_8690函数同样存在反调试：

其中的sub_85e0函数会对签名作校验，并且开辟空间生成一段新的代码，通过((void (__fastcall *)(int *))((char *)v8 + 1))(&v9)进行调用，变量v9即为用户输入的字符串。

动态调试

我用的是Android Stdio的模拟器，userdebug版本。

第一步

把ida的dbgsrv目录下android_server文件放到手机的目录

adb push android_server /data/local/tmp

赋予可执行权限，并以root用户运行android_server

chmod 777 android_server

第二步

adb进行端口转发

adb forward tcp:23946 tcp:23946

第三步

以调试模式启动应用，这样程序会在java层代码停下来

am start -D com.a.sample.loopcrypto/.MainActivity

包名可以通过Apktool解压查看AndroidManifest.xml得到，如果直接解压apk压缩包，打开xml会看到一堆乱码。

IDA调试器选择Remote ARM Linux/Android debugger，并设置Debugger options为加载动态库时挂起：

通过Attach to process附加到目标进程：

第五步

Android Stdio选择Attach debugger to Android process：

patch反调试

一切准备完毕，开始运行IDA，程序停在了加载base.odex库的位置，继续运行，停在了加载libcheck.so库的位置，ctrl+s查看libcheck.so库的基址：

加上init_array里的函数地址偏移，下断点：

恢复运行，程序会在断点处停下来，因为整个函数除了反调之外没有其他作用，所以我们直接修改PC到return地方即可。为了保证堆栈平衡，我们需要执行完push和pop指令，所以执行完偏移为83DC处的push指令，将PC指向偏移为8440处的pop指令：

在check函数下断点（偏移为87FC），继续执行，在模拟机上随便输入几个字符串，IDA会在check函数停下来，跟进来到第二处反调试，同样通过修改PC绕过反调试：

执行到地址E3A836B0处，修改PC为ptrace的下一条指令即可。然后继续单步执行，来到动态生成的代码段：
